# 🏏 Score Guru

A premium, mobile-first **live cricket scoring** app for turf cricket — in the spirit of CricHeroes/Stumps, scaled down for a crew of friends. Ball-by-ball scoring, gorgeous IPL-style scorecards, realtime live viewing, and all-time + monthly leaderboards.

- **Public layer** (no login): branded home, live match (auto-updating), past scorecards with ball-by-ball over log, player profiles, stats/leaderboards, and a "submit a player" form.
- **Admin layer** (single password): approve players, manage teams, create matches, run the toss, score ball-by-ball, edit match details, delete matches.

Built with **Next.js (App Router) + TypeScript**, **Tailwind CSS**, **Framer Motion**, and **Supabase** (Postgres + Realtime + Storage). Deploys to **Vercel**.

---

## How it works (architecture)

The whole app rests on one idea: **a match is an immutable event-log of deliveries.** Everything else — live score, strike rotation, scorecards, partnerships, fall of wickets, economy, leaderboards, Player of the Match — is **derived** by replaying that log (`src/lib/cricket/engine.ts`). This gives:

- **Perfect undo** — just delete the last event and re-derive.
- **Always-accurate stats** — no cached totals to drift out of sync.

### Security model

"Simple login" without being insecure:

- The admin enters **one password**, verified **server-side** (`src/lib/auth.ts`) against `ADMIN_PASSWORD`. On success the server issues a **signed, httpOnly** session cookie (JWT via `jose`).
- **Every write** (matches, deliveries, approvals, deletes) goes through **Next.js Server Actions** that call `requireAdmin()` and use the Supabase **service-role key** (server-only env var).
- The browser only ever gets the Supabase **anon/publishable key**. Postgres **Row Level Security** allows public `SELECT` only (plus a constrained public *insert* for player submissions, forced to `status='pending'`). No public updates/deletes. Scores are tamper-proof.

## Local setup

### 1. Prerequisites
- Node.js 18.18+ (Node 20 recommended)
- A free [Supabase](https://supabase.com) project

### 2. Install
```bash
npm install
```

### 3. Create the database
Open your Supabase project → **SQL Editor** → paste the contents of [`supabase/migration.sql`](supabase/migration.sql) and run it. This creates all tables, indexes, enums, the shared event sequence, RLS policies, storage buckets (`player-photos`, `team-logos`), and adds the live tables to the Realtime publication.

> The migration is idempotent — safe to re-run.

### 4. Environment variables
Copy `.env.example` → `.env.local` and fill in:

| Var | Where to find it | Exposed to browser? |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase → Project Settings → API → Project URL | ✅ |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Project Settings → API → `anon`/publishable key | ✅ |
| `SUPABASE_SERVICE_ROLE_KEY` | Project Settings → API → `service_role` key | ❌ server-only |
| `ADMIN_PASSWORD` | You choose it | ❌ |
| `SESSION_SECRET` | `openssl rand -base64 48` | ❌ |

### 5. Run
```bash
npm run dev
```
Open http://localhost:3000. Admin console is at `/admin` (you'll be redirected to `/admin/login`).

## First match in 5 steps
1. **Teams** (`/admin/teams`) — create two teams (name, colour, optional logo).
2. **Players** (`/admin/players`) — add players directly, or approve ones submitted via `/submit-player`.
3. **New match** (`/admin/matches/new`) — pick overs, toggles (free hit / last-man-stands / block-consecutive-overs), the two teams, and build each lineup (a player can't be on both).
4. **Toss** — record who won and bat/bowl. Match goes live.
5. **Score** — tap runs/extras/wickets. Live viewers at `/live` update instantly. At the end the result + Player of the Match are computed automatically.

## Deploy to Vercel
1. Push this repo to GitHub.
2. Import it in Vercel.
3. Add the five environment variables (above) in **Project → Settings → Environment Variables**.
4. Deploy. (No build config needed — it's a standard Next.js App Router app.)

### PWA
The app is installable (manifest + icons in `public/`). On mobile, "Add to Home Screen" gives a full-screen, branded experience. Icons are generated by `scripts/gen-icons.mjs` (run with a one-off `npm i -D sharp` if you want to regenerate them).

## Conventions
- Overs shown as `4.3` (4 overs, 3 balls). Economy = runs ÷ overs. SR = runs ÷ balls × 100.
- Timezone **IST**; dates **dd/mm/yyyy**.
- Super-over runs are **excluded** from career stats and leaderboards.
- A player who has appeared in a match is **hidden** rather than hard-deleted (preserves historical scorecards).

## Project layout
```
src/
  app/                 # routes (public + /admin)
  components/          # ui, public, match, admin components
  lib/
    cricket/engine.ts  # ★ derive live state + scorecards from the delivery log
    cricket/stats.ts   # career aggregation, leaderboards, POTM
    cricket/matchview.ts
    actions/           # server actions (all writes, admin-gated)
    auth.ts            # password verify + signed session cookie
    supabase/          # server (service-role) + browser (anon) clients
supabase/migration.sql # full schema + RLS + storage + realtime
```
